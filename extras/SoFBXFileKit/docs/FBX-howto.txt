To avoid surprises, to this on a Mac

Basic FBX && Coin setup:

o Download and install the FBX SDK from Autodesk:
  http://usa.autodesk.com/adsk/servlet/index?siteID=123112&id=6837478
o Check out and build the foreignfiles branch of Coin:
  https://svn.coin3d.org/repos/Coin/branches/foreignfiles
o Build/Install Qt4 && SoQt
o Build SoFBXFileKit:
  mkdir Coin/extras/SoFBXFileKit-config
  cd Coin/extras/SoFBXFileKit-config
  export FBXSDKDIR=/Applications/Autodesk/FBXSDK20093
  export COIN_DEBUG_FBX=1
  cmake -DCMAKE_INSTALL_PREFIX=/Users/kintel/install/Coin -DCMAKE_BUILD_TYPE=debug ../SoFBXFileKit
  make install
o Symlink the SoFBXFileKit library from Coin's library search path. On Mac:
  cd /Library/Frameworks/Inventor.framework/Libraries
  ln -s $HOME/install/Coin/lib/libSoFBXFileKit.dylib
o Test examples:
  cd Coin/extras/SoFBXFileKit/examples
  ./fbxviewer <file.fbx>
  ./foreignconvert <file.fbx> (converts to an .iv file in the same dir as the fbx file)

Test system setup:

o Check out and build CoinSurance:
  mkdir CoinSurance-config
  cd CoinSurance-config
  export COINFBXDIR=$HOME/install/Coin
  export FBXSDKDIR=/Applications/Autodesk/FBXSDK20093
  cmake ../CoinSurance
  make 
o Run tests
  ctest -R write
o The following tests will fail due to known issues:
	162 - compare_write_bump-transtex (Failed)

	198 - compare_write_subobject-max (Failed)
